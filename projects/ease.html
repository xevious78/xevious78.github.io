A bit of Context

Atos IT Challenge

The Atos IT Challenge is an annual IT event that gathers students from all over the world. This contest rewards the team with the most innovative and brilliant concept that solves a current problem.


2014's Edition was about improving people's life using connected devices. 60 teams took part in this challenge.

Analysis

We began our projet with an in-depth analysis of the smart appliance market. This led us to the conclusion that the Smart Home was a myth.

No ecosystem

Smart devices are made by various manufacturers, each of which has developed its own dedicated software to control, monitor and analyse the system. However, those software are not made to communicate with each other.
If the user wants to live in a true Smart Home, she has to buy her devices from the same manufacturer.

The market lacks from a solution which allows various devices to communicate together in order to create new use cases and to eventually push the user to buy those kind of appliances.

Energy consumption is a matter of habits

Many devices are available to monitor in real-time a family energy consumption. However, this kind of system is not able to correlate the consumption to an actual in-house event.
How can somebody improve the energy efficiency if there is no way to know exactly which habit should be modified ?

The market lacks from a system that analyses the user's habits in order to provide her with precious advice.

Intelligent Workflows

EASE provides the family with customised and intelligent workflows for any house-holding tasks. It takes into account various parameters in order to generate the perfect set of tasks.

Once a workflow is validated, EASE can trigger and control each device on time based on its dedicated schedule. EASE can also react to external events by tweaking the schedule in real-time.




EASE was among the 15 ideas selected by the jury.

We had 3 months to develop of working Proof of Concept.

Development

I was the Tech Leader of this project: I managed the development and made sure everything was finished in time.
During this phase, we mainly focus on the Workflow Generator system.

Pipeline

The user query is first processed thanks to a Natural Language Processing module.
Then, EASE seeks for abstract recipes to perform the query.
EASE selects the devices and the family members that can do the tasks required by the recipe.
Finally, EASE schedules each task so that the query can be finished in time.

Once a set of tasks is validated, EASE is able to trigger each device on time.

Official Review (Download)
Look & Feel: 4/5
Technical: 4/5
The concept of the app is really good, it is scalable for other home uses (not only for cooking) and it works in real time with the server side with sockets for the tasks.
The app is not robust and ready for app store because there are some minor UI mistakes (labels with null values, menu options without interaction, etc.) and it can be crash easily with multitouch, but for a normal use of the app it responds successfully. The main negative point is that it only support iOS 8, ignoring near to a 20% of the users.
The overall work done is very good. The iOS app presents a high level of complexity, using a lot of different views, many REST service calls, and follows modern guidelines. The frameworks and libraries used give a lot of value, the developers took good profit of them to solve common problems found as well in real production apps (login / JSON fields validations / creation of new items / lateral menus / animated transitions / text recognition and interpretation).
Also the documentation provided (text documents, Postman collections) is quite complete and easy to follow, to set up all the environment needed. Intuitive and easy to use. Just needs more contrast between text and background color. Great!
